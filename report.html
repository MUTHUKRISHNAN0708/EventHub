01. Abstract

The EventHub project represents a state-of-the-art solution for modern event planning, designed to bridge the gap between premium service providers and high-expectation clients. By leveraging the MERN (MongoDB, Express, React, Node.js) stack, the platform facilitates seamless vendor discovery, transparent booking management, and robust administrative oversight.

Key innovations include a custom "Glassmorphism" UI design system, a highly resilient database connection architecture capable of surviving transient network instability, and a performant search engine optimized through advanced MongoDB indexing. This report details the architectural decisions, implementation workflows, and rebranding strategies that transformed a concept into a secure, scalable, and visually stunning digital product.

Through iterative development and rigorous optimization, EventHub has achieved near-zero latency in data retrieval and 100% reliability in cloud-native deployments, setting a new standard for local-to-cloud development lifecycles.

03. Technology Stack

Frontend Architecture

  • React 18: Component-driven reactivity.
  • Vite: Optimized build pipeline.
  • Tailwind CSS: Rapid UI styling.
  • Lucide Icons: Consistent vector assets.

Backend & Data

  • Node.js: Non-blocking event loop.
  • Express: Minimalist web framework.
  • MongoDB Atlas: Multi-region database.
  • JWT: Encrypted token security.

The synergy between these technologies allowed for a "One Language" (JavaScript) full-stack development. This improved development speed and allowed for tighter integration between client-side state and server-side validation logic.

Furthermore, the use of Mongoose as an Object Data Modeling (ODM) layer provided strict schema enforcement while maintaining the flexibility of a NoSQL database structure.

04. Database Systems

The EventHub database is engineered for rapid filtering and detailed relational linking in a non-relational environment. We utilized MongoDB Atlas for its distributed nature, ensuring that the platform remains available regardless of regional datacenter outages.

The schema relies on four critical intersections:

  • Vendor-Service Mapping: One-to-many relationship using optimized ObjectIDs.
  • User-Booking History: Secure tracking of transactional metadata.
  • Availability Ledger: Real-time lookup of venue dates for conflict prevention.
  • Index Tuning: Custom compound indexes on high-traffic fields.

By shifting to an Index-First design, we reduced searching time for specific event categories (like "Premium Wedding Photography") by over 80% compared to standard scanning methods.

06. Backend Resilience

The backend of EventHub was hardened against network fluctuations. Specifically, we addressed SRV lookup ETIMEOUT errors by implementing a robust reconnection strategy.

Engineering Solutions

  • DNS Hardening: Overriding local DNS resolution to utilize Google Public DNS (8.8.8.8).
  • Connection Heartbeats: Monitoring socket health at sub-30s intervals.
  • Smart Listeners: Delaying the server 'on' state until a verified database handshake is complete.

These improvements were verified through rigorous stress tests, ensuring the API remains authoritative even during localized internet outages or DNS propagation delays.
